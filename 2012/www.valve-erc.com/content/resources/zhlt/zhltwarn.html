HLCSG
map.c degenerate QuArK-style brush texture
Send email to the author if this occurs
map.c Entity %i, Brush %i: origin brushes not allowed in world
This is caused by having an brush with the ORIGIN texture not part of any entity. The brush will be ignored by the tools.
textures.c fseek to %d in wadfile %s failed\n", wadinfo.infotableofs, pszWadFile);
Send email to the author if this occurs
textures.c ::: Unterminated texture name : wad[%s] texture[%d] 'name'[%s]
The wad file listed is corrupt. This was determined because the field for the name of a texture was longer than the limit, which could either be just random garbage, or other problem.
textures.c ::: Larger than expected texture (%d bytes) Wad:wad[%s] Texture[%d:%s]
The WAD files store the size of each texture (plus some of its mipmap levels). If this value is larger than the largest texture (256x256+ its mipmaps) this is a sign the wadfile is corrupt or has non-standard textures.
textures.c ::FindTexture() texture %s not found in BSP's wad file list!
Pretty self-explanitory, a texture could not be found in any of the wad files.
textures.c fseek to %d failed
Send email to the author if this occurs
textures.c ::LoadLump() texture %s not found in BSP's wad file list!
Pretty self-explanitory, a texture could not be found in any of the wad files.
textures.c no wadfile specified
The .map file did not contain a "wad" or "_wad" attribute listing all the wadfiles it expects to use.
wadinclude.c WadInclude file %s does not exist
Send email to the author if this occurs

HLBSP
outside.cg_hullnum %i: No entities in empty space -- no filling performed"
There is a point entity in the 'void'. All entities are required to be 'inside'
outside.cLEAK LEAK LEAK
Fix your leak or you will be unable to run hlrad properly
qbsp.cMax subdivide size is %i, '-subdivide %i' ignored
The -subidivide command line paramter was passed a value larger than the maximum possible value, and subsequently set to the maximum value.
qbsp.cMin subdivide size is %i, '-subdivide %i' ignored
The -subidivide command line paramter was passed a value smaller than the minimum possible value, and subsequently set to the minimum value.
solidbsp.c::ChoosePlaneFromList() surface fragmentation undesireability exceeded WORST_VALUE
Send email to the author if this occurs
solidbsp.cMakeNodePortal:new portal was clipped away from node@(%.0f,%.0f,%.0f)-(%.0f,%.0f,%.0f)",
A vis portal was created but ended up not being used, probably worth fixing as the side effects can vary from minor to strange.
surfaces.cSubdivideFace: didn't split the %d-sided polygon @(%.0f,%.0f,%.0f)",
The brush at the coordinates listed was fragmented (or perhaps designed) to have many verticies on a single face. Some of the other brushes intersecting this one tried to split it beyond the vertex limit.
tjunc.cCanonicalVector: degenerate
This warning should include some coordinates. Go there, and simplify whatever complex objects are in the area.

HLRAD
lightmap.clight at (%i %i %i) has missing target
A light_spot sytle light is missing a target to point at. It will point to the center of the map (0,0,0) without this information.
lightmap.cToo many direct light styles on a face(%f,%f,%f)
A few dynamic lights in an area are trying to light up the same surfaces. This limit is 4. Any light with a targetname (whether it was meant to be switchable or not), and any light set to 'pulsating' etc counts against a dynamic light in the area.
qrad.cignoring bad texlight '%s' in %s
The .rad file listed has a bogus entry. The format is texturename red green blue brightness
qrad.cDuplication of texlight '%s' in file '%s'!
The .rad file listed has multiple entries for the same texture.
qrad.cOverriding '%s' from '%s' with '%s'!
Multiple .rad files had entries for the same texture, with different values.. This is just a warning signifying this has happened, and showing which file ended up having priority.
qrad.cRedundant '%s' def in '%s' AND '%s'!
Multiple .rad files have entries for the same texture, with the same value.
qrad.ctransfer ~= 0 : scale=(%f) dist=(%f)
Send email to the author if this occurs
qrad.cSwapTransfers: unmatched (light patch with no surface area)
Send email to the author if this occurs
qrad.cMissing transfer count! Save file will now be rebuilt.
The files generated with a previous run of -dump with hlrad could not be recovered and hlrad had to reconstruct the data.
qrad.cMissing patch count! Save file will now be rebuilt.
The files generated with a previous run of -dump with hlrad could not be recovered and hlrad had to reconstruct the data.
qrad.cIncorrect transfer patch count found! Save file will now be rebuilt.
The files generated with a previous run of -dump with hlrad could not be recovered and hlrad had to reconstruct the data.
qrad.cNo vis information, direct lighting only.
HLRAD was run without HLVIS being run first. Only direct lighitng can be done, which is equivalent to running -bounce 0 when there is vis data normally. This warning commonly occurs when compiling from within Worldcraft in normal mode instead of expert mode, as it does vis after rad for some inane reason.
