Utilities
Everything you need to get you going.

Valve Hammer Editor
The first place you need to start is the editor. Formerly known as Worldcraft, the Valve Hammer Editor is your main and best choice for Half-Life and Half-Life mod mapping. (I'm a little biased, ya think? ) Hammer 3.4 is the newest version. It requires that you have a video card capable of rendering in OpenGL.

  • Valve Hammer Editor 3.4 [Half-Life, Half-Life mods]

Compile Tools
When it comes to compile tools, there really isn't a choice. Well, there is. You can use the default tools that come with Hammer, or you can use the good stuff. Zoner's compile tools have undergone major surgery and emerged stronger, smarter, and faster. They've got cool features too.

  • Zoner's compile tools [Half-Life, Half-Life mods]

Game Data Files
The game data files are what Hammer uses to to interface with a game's entity data. You can find a large assortment of game data files in the FGD project at DDR. If you are making maps for a mod that is not listed in the FGD file section, you might try contacting the mod authors themselves.

The PAK File
In Half-Life, a majority of the content was stored in the PAK file. A PAK file is similar to a ZIP file in that it stores a collection of files, maintaining all of it's directory information. Often it is handy to browse or extract things from the PAK file for modification or so other programs can make use of the content -- Hammer, for example.

A number of PAK browsing and editing utilities exist, but the two best ones to have are Wally and PakScape. PakScape is a nice simple PAK browser and editor. Wally is discussed below and, as you'll see, is the preferred tool.

  • PakScape
  • Wally

Textures/WADs
There are two tools that you should have if you want to make or modify game textures - Paint Shop Pro, and Wally.

Paint Shop Pro, for those unfamiliar with it, is a feature rich image editor from Jasc software. A 30 day trial version is available, and for anyone doing serious texture work, this is a great alternative to the more expensive PhotoShop.

Wally is primarily a texture editor. You can load in images created by other programs (like Paint Shop Pro) or create them within Wally. Wally also has a number of features (mipmapping, decal wizards, etc) that are specific to game texture creation. Wally also functions quite well as a PAK file editor. To top it all off, Wally is freeware.

  • Paint Shop Pro
  • Wally

Sprites
If you're creating sprites or need a tool to view sprites, the following programs will come in very handy.

SprView is a program by David Flor that lets you view Half-Life sprite files. In addition, you can save out their individual bitmap files. This is useful if you want to modify existing sprites or see how they were made.

The Sprite Wizard is a sprite creation tool that walks you through the process of making a sprite.

  • SprView (direct download)
  • Sprite Wizard (direct download)

Models
For looking at or editing/creating models, there is really only one place you need to go: chUmbalUm-sOft. If you only need to view models, the Half-Life Model Viewer will get the job done well. It's an OpenGL accelerated mdl viewer that allows you to view all of a models animations and different model settings (guns, etc). You can also export and import skins, allowing you to give models a custom look.

If creating or editing models is your thing, you'd do best to check out MilkShape3D (which, I should add, has nothing at all to do with shaping milk). This program has all the features necessary to edit or create models for Half-Life and many other games. If you're using 3DS Max 3.1 to create models, you'll also need the newer model exporter plugin, Smdlexp.dle, linked below.

  • Half-Life Model Viewer (direct download)
  • MilkShape3D (direct download)
  • Smdlexp.dls - model exporter for 3DS Max 3.1

The SDK
If you plan on writing up a mod, you'll need the game code aka the Half-Life SDK. The Half-Life SDK comes in two versions -- full and source-only. If you do not require any example material, the source-only version is recommeneded, as it's much smaller than the full install. Full information can be found at the link below.

  • Half-Life SDK Project home

For information and tutorials on coding, see the Handy Vandal's coding section.

Miscellaneous
Below are some miscellaneous utilities which you might find useful.

GenSurf
GenSurf, by David Hyde, is a terrain generator for Quake-based games, including Half-Life of course. This tool makes the creation of rolling terrain, rocky cliffs, etc, much much easier. When combined with Zoner's compile tools (above), GenSurf can even make use of hint brushes in Half-Life.

  • GenSurf (local download)

Tool Box Compilation Controller
Sometimes (very often in fact) it is more efficient to compile your maps through something other than Hammer. Doing this can be a daunting task to those only familiar with the GUI environment of Windows. Batch file is just a meaningless phrase to most. Enter the niche of the compiler frontend.

  • Tool Box Compilation Controller - The Tool Box Compilation Controller (TBCC) acts as a GUI frontend to all the command line parameters that are involved when compiling a level. It's pretty intuitive to setup and use and includes some powerful features.
  • Batch Compiler - Another very promising compiler frontend, this one is specifically tailored to Zoner's compile tools. This is definitely worth checking out.

If you want to use a batch file, however, there is a simple way to create one customized to your personal settings. Simply use the Compile Batch File script to generate one. If you go this route, it's recommended that you be using Zoner's compile tools.

If you've come across other utilities which you think should be on this list, drop me a line and I'll check it out!
