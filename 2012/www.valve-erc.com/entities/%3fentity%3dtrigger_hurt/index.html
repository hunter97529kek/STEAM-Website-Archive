trigger_hurt
Half-Life entity

description

    The trigger_hurt entity can cause pain and damage.

usage

    It has the following properties:

    • Name (targetname) - If a targetname is specified, the trigger_hurt can be triggered on and off.
    • Target (target) - Each time the trigger_hurt is "activated", it can trigger an event. "Activated" here means each time it causes damage.
    • Master (master) - You can specify a multisource master here.
    • Damage (dmg) - This is the amount of damage the trigger_hurt area will cause.
    • Delay before trigger (delay) - If the trigger_hurt targets an event, the delay setting will delay the activation of the event.
    • Damage Type (damagetype) - You can specify the type of damage here. Specific damage types are:

      • 0 (GENERIC)
      • 1 (CRUSH)
      • 2 (BULLET)
      • 4 (SLASH)
      • 8 (BURN)
      • 16 (FREEZE)
      • 32 (FALL)
      • 64 (BLAST)
      • 128 (CLUB)
      • 256 (SHOCK)
      • 512 (SONIC)
      • 1024 (ENERGYBEAM)
      • 16384 (DROWN)
      • 32768 (PARALYSE)
      • 65536 (NERVEGAS)
      • 131072 (POISON)
      • 262144 (RADIATION)
      • 524288 (DROWNRECOVER)
      • 1048576 (CHEMICAL)
      • 2097152 (SLOWBURN)
      • 4194304 (SLOWFREEZE)


    Flags
    There are a number of flags available:

    • Target Once (1) - If the trigger_hurt targets an event, it will activate that event each time it "hurts" something. If this flag is set, it will only activate the event the first time it "hurts" someone.
    • Start Off (2) - This flag allows you to start the trigger_hurt in off mode, then trigger it on.
    • No Clients (8) - If this flag is set, the trigger_hurt wont affect any clients (players).
    • FireClientOnly (16) - The trigger_hurt will hurt anyone who touches it but only fire its target when it actually hurts a client (player).
    • TouchClientOnly (32) - Only the client (player) can be hurt by this trigger.
