weapon_displacer
Opposing Force entity

description

    The Displacer can be used to teleport creatures away from you or to teleport yourself out of trouble.

usage

    Below are this entity's properties.

    • Name (targetname) - Giving this object a name will allow it to be affected by other entities.
    • Target (target) - When this object is picked up, it can activate an event if the name of the event is specified here.
    • KillTarget (killtarget) - When this object is picked up, it can kill an event if the name of the event is specified here.
    • Delay before trigger (delay) - If a value is specified here, the Target or KillTarget action will only be performed after the specified value in seconds.
    • Render FX (renderfx) - See Rendering Properties.
    • Render Mode (rendermode) - See Rendering Properties.
    • FX Amount (renderamt) - See Rendering Properties.
    • FX Color (rendercolor) - See Rendering Properties.

    Flags
    This entity has one flag.

    • Not in Deathmatch (2048) - Setting this flag will make the entity not appear in deathmatch play.

    Earth/Xen Self Displacement
    In multiplayer, the displacer is very straightforward from a level design standpoint. Firing the primary mode at a player will either teleport them on a direct hit, or kill them with splash damage or an indirect hit. Secondary fire mode causes the player to teleport to a random deathmatch spawn point but fires a displacer ball directly where the player just teleported from, damaging or killing anything in that area.

    For single player, the displacer needs certain entities present in the level to allow self displacment.

    The primary fire mode for the Displacer will always work if the player has sufficient ammunition. For the secondary fire mode to teleport the player to Xen, the player must have sufficient ammunition, and the level that the player is in must have certain entities.

    In Half Life: Opposing Force single player, the maps that are present after the player receives the Displacer contain 2 parts. A main part which is typically "Earth" and a small mini-Xen area. These are both contained within the same level (.bsp).

    The first entity that must be placed is an info_displacer_earth_target. This is typically placed in the Earth part of the map. You need at least one of these in your level. Place this entity at the point at which you want the players feet to be when he returns. When placing multiple info_displacer_earth_target entities here is how the game treats these entities. When the player self displaces, the game records what the closest info_displacer_earth_target is, and it is that target to which the player will return to from Xen.

      Note: It is very easy to "break" maps with info_displacer_earth_target entities. Be careful with their placement, or you may find that players will return from Xen having skipped over a good portion of your level.

    The second entity that needs to be placed is an info_displacer_xen_target. Only place one of these entities. This entity is placed in the mini-Xen area and is the point at which the player self-teleports to with the displacer.

    These are the only entities needed to make the Displacer secondary fire mode actually teleport the player, BUT how does the player get back from Xen? If the level designer wishes to have the player return to Earth from Xen, then one more brush based entity called the trigger_xen_return is needed. When the player enters the area that this trigger brush occupies, they will be teleported to the info_displacer_earth_target that was the closet to their location upon self teleport.

    Notes
    The code automatically adjusts for Earth and Xen gravity depending upon which entity you are traveling to. This eliminates the need for placing additional entities at each displacer target such as trigger_gravity entities.

    Avoid giving the player any ammo_gauss (uranium) ammunition while they are in Xen. Here is the reason:

    The player can only carry a maximum of 100 cells. Self displacing takes 60 cells, leaving the player in Xen with only 40 cells, they cannot self displace again, this is what we want. If they pick up cells in Xen, and self displace they will just go back to the info_displacer_xen_target. This is fine, BUT, what happens is that the Displacer code goes through the routine of finding the closest info_displacer_earth_target again. Depending where the mini-Xen level is placed in the world in relation the the "Earth" portion of the map, this will most likely break the single player level, returning the player to some point in the map that the player should not be.

    Of course with careful planning and testing, this can be overcome, but keep this in mind when creating a single player level that allows self displacement.
