func_tank
Half-Life entity

description

    Use this to create a controlable gun turret.

usage

    The gun should be created facing angle 0 (in the top view, it should point to the right). Use the angle indicator to specify which direction it should face in the game.

    1. The first step is, of course, to build the brush model of the gun. It can be anything you like, although normally you'd want it to look like some type of gun. It is important to note that an origin brush must be included in the construction of the gun.

        An origin brush is a brush covered in the ORIGIN texture. Any entity that rotates absolutely *must* have an origin brush as part of their makeup.

        Note: the func_tank brush model must be built facing east (to the right in the top view). Later, when you turn it into a func_tank, you will set the actual facing of the gun in the game by setting the angle value of the entity.

    2. Turn the brush model into a func_tank. Select it, and setup its properties.

      • Name (targetname) - The gun must have a unique name. This will be used to control the gun through the func_tankcontrols (see below).
      • Target (target) - The Name (targetname) of an ambient_generic should be specified here. This will be the sound the gun makes when fired.
      • Yaw Rate (yawrate) - This is the speed of the left/right movement of the gun. For a fast feeling, a value of 600 is good.
      • Yaw Range (yawrange) - This is the range of left/right movement in degrees.
      • Pitch Rate (pitchrate) - This is the speed of the up/down movement of the gun. For a fast feeling, a value of 600 is good.
      • Pitch Range (pitchrange) - This is the range of up/down movement in degrees.
      • Barrel Length (barrel) - This is the length from the center of the origin brush to the tip of the barrel.
      • Barrel Horizontal (barrely) - This is the horizontal distance from the center of the origin brush to the center of the barrel.
      • Barrel Vertical (barrelz) - This is the vertical distance from the center of the origin brush to the cetner of the barrel.
      • Flash Sprite (spriteflash) - This is the sprite that will be appended at the end of the barrel when the gun is fired, sprites/muzzleflash2.spr for example.
      • Sprite Scale (spritescale) - Use this to set the scale of the sprite, if the default size is not to your liking.
      • Rate of Fire (firerate) - This is the rate of fire per second.
      • Damage per Bullet (bullet_damage) - Set the bullet damage here.
      • Persistance (persistance) - This is the firing persistance of the gun. This is really only useful for non-user controlled turrets (see below). This will make the gun continue to fire for a moment after the player leaves its visibility range, rather than being instant on/off.
      • Bullet Accuracy (firespread) - Set the bullet accuracy here.
      • Bullets (bullet) - Set the bullet type here.

    3. You will also need to set a flag in the Flag properties of the func_tank. For a user controllable turret, all that needs to be checked is the Controllable flag.
    4. Create a func_tankcontrols entity just behind the func_tank. Set it to target the Name (targetname) of the func_tank.
    5. Place a clip brush over the func_tank so the player can't accidentally interfere with its movement. This may require a bit of tweaking until it is just right.

        A clip brush is a brush covered in the CLIP texture. It is invisible in the game, but blocks player movement. Ammo *will* pass through a clip brush, however

    6. If you haven't created the ambient_generic that is targeted by the func_tank yet, now is the time to do it. A good sound to use for a gun is the ambience/biggun1.wav sound. In the flags section of the ambient_generic properties, set its radius, and make it 'Start Silent' and 'Not Looped'.


    Flag Properties
    There are a number of flags that can be set for this entity.

    • Active - this will make the gun start active. It is useful for making monster controlled turrets. This is unneeded for user controlled turrets.
    • Player - for monster controlled turrets, this will cause the gun to fire at the player.
    • Humans - for monster controlled turrets, this will cause the gun to fire at humans.
    • Aliens - for monster controlled turrets, this will cause the gun to fire at aliens.

        Note: The above three flags were not implemented. They will be removed from subsequent versions of the Half-Life game data files.

    • Only Direct - only fire if target is in direct line of sight.
    • Controllable - this is used to designate this gun as user controllable.

    Monster Controlled Turrets
    In various places in Half-Life, you will have seen gun turrets that are being controlled by military grunts. These are quite simple to create once you have the standard func_tank created.

    1. Create the func_tank, as above.
    2. Select the func_tank and press Alt+Enter to bring up its properties. Click on the Flags section. The only flags you want checked are 'Active' and 'Player'.
    3. A monster controlled func_tank does not require a func_tankcontrols entity, so if one is present (and connected to this func_tank) remove it.

      The above are the only diffences in the func_tank, between a normal user controlled func_tank, and a monster controlled version.

    4. Now, place the controller. In this example, we'll use a monster_human_grunt. Place it in a position just behind the gun.
    5. Select the monster and press Alt+Enter to bring up its properties. In the TriggerTarget field, enter the Name (targetname) of the func_tank. In the Trigger Condition field, select "Death".
    6. In the monster's Flags properties, place a check beside the Prisoner flag. This will make the monster stay by the gun (making it look like he is controlling it). Otherwise, the monster will run at you and the gun will continue to fire.


    When this is placed in the level, the grunt (well, the turret) will fire at the player whenever he has site of him. Killing the grunt will disable the turret.
