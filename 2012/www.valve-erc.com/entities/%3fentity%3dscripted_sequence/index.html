scripted_sequence
Half-Life entity

description

    The scripted_sequence entity allows you to make a monster perform a pre-scripted animation.

usage

    It can also be used to simply make a creature go to a specific location. Below are the steps to make a simple animated sequence: a scientist that will wave when a trigger is passed through.

    • Name (targetname) - This entity can be activated by another trigger. If this is what is being done, a name must be entered here.
    • Target (target) - This event will be activated when the script is complete.
    • Killtarget (killtarget) - This event will be removed from the game when the script is complete.
    • Delay (delay) - The above two properties will be delayed by this value in seconds, after the script has completed.
    • Target Monster (m_iszEntity) - This is the Name (targetname) of the monster this script will affect. If you want the script to act with any monster of a certain type, specify the monster type here instead (ie: monster_scientist).
    • Action Animation (m_iszPlay) - This is the animation that will be performed by the targetted monster.
    • Idle Animation (m_iszIdle) - This is the animation the monster will return to after performing the Action animation.
    • Search Radius (m_flRadius) - The script will look for a valid monster in this radius. Set this to 0 if you want the script to be triggerable only.

        Note: If a Search Radius (m_flRadius) is specified, the scripted_sequence will automatically search the specified radius in a period equal to the Repeat Rate ms (m_flRepeat) value. If you only want this scripted_sequence to be triggerable, give the Search Radius a value of 0.

    • Repeat Rate ms (m_flRepeat) - The above search will be performed every n seconds, where n equals the value you enter here.
    • Move to Position (m_fMoveTo) - The value of this determines how the monster moves in relation to the aiscripted_sequence entity. The allowable values are:

      • No (0) - The monster will perform its animation sequence wherever it is.
      • Walk (1) - The monster will walk to the location of the aiscripted_sequence, then perform its animation.
      • Run (2) - The monster will run to the location of the aiscripted_sequence, then perform its animation.
      • Instantaneous (4) - The monster will warp instantly to the location of the aiscripted_sequence, then perform its animation.
      • No - Turn to Face (5) - The monster will turn to face the location of the aiscripted_sequence, then perform its animation.


    Flags
    There are a number of flag properties that you can set to effect the behavior of the scripted sequence.

    • Repeatable (4) - This lets the sequence be triggered more than once.
    • Leave Corpse (8) - If a character dies in a script with this flag set, it's
      animation is frozen in the last position, and a "corpse" is left. Otherwise, dead characters will fade out.
    • No Interuptions (32) - This means that the script will not stop for any reason, even if
      the player shoots the character, until the script completes.
    • Overide AI (64) - Sometimes a character's AI will keep them from playing a script because the character is fighting or something like that. We placed a bunch of scripts for little things like an "interesting" computer panel, or some atmospheric detail like that. So, unless this flag is set, the AI can feel free to ignore sequences if it's doing something important.
    • No Script Move (128) - This disables the code that tries to fixup the position of a character at the end of a script so the AI can properly take over. Only turn this on if the character jumps to a new position after playing the
      script (and there are other reasons why that can occur such as not setting up the animation correctly in the model's .qc file)

    Example of Use
    Following is a brief description of how to make a scientist wave on demand.

    1. Place a monster_scientist in you map, and give him a 'targetname' of 'John'.
    2. Place a scripted_senquence in the map, and give it a 'targetname' of 'wave'.
    3. Where the scripted_sequence asks for a Target Monster (m_iszEntity) enter the name of the monster you want to use this scripted_sequence, in this case 'John'.
    4. For the Action Animation (m_iszPlay) enter the name of the action animation for the particular monster you are using. For the waving scientist, you would use 'wave'. A list of animations available to scripted_sequences can be found in the Half-Life entity guide (which itself is accessible through the Help menu in Worldcraft).
    5. Last, where it says Move to Position (m_fMoveTo), you have a choice between: No (0), Walk (1), Run (2), Instantaneous (4), and Turn to Face (5). Select 'Walk' for this example.
    6. Place a trigger_once that targets 'wave'. Make sure the angle of the scripted_sequence is aimed at the trigger_once area. When the player walks through the trigger_once, the scientist will turn to the scripted_sequnce area, turn towards the trigger area, and wave.
