env_laser
Half-Life entity

description

    The env_laser allows you to setup damage-causing beam effects. It is very similar to the env_beam entity.

usage

    Below are it's properties.

    • Name (targetname) - This entity can be triggered on and off. If you give it a Name, it will start off unless you enable the Start On flag (see below).
    • Target of laser (LaserTarget) - The actual laser will extend from the env_laser location to the object specified by this value.

        Note: Generally the laser target will be an info_target entity, but you can use other entities. Anything that can have a name can be a target. If you are going to use a non-point entity as a target (a func_wall, for example), you need to include an origin brush as part of the entity. An origin brush is a brush that is covered with the texture named ORIGIN, and it will be invisible in the game. The exact center of the origin brush will be used as the laser target.

        If you have several objects with the same name as the laser target, the env_laser will jump from point to point randomly. However, the laser will always originate at the env_laser location.

    • Render FX (renderfx) - See rendering properties for information on Render FX.
    • Brightness (1 - 255) (renderamt) - This sets the brightness of the beam. It acts the same way transparency does - a higher number equals a more "solid" beam.
    • Beam color (R G B) (rendercolor) - Use an RGB value to set the color of the beam.
    • Width of beam (width) - This can be a value between 0 and 255. In game, it will equal (n * 0.1) pixels in width.
    • Amount of noise (0 - 255) (NoiseAmplitude) - The amount of noise determines how "active" the laser beam will be. This is used to simulate the arcing movement of an electrical beam. A noise value of 0 will produce a flat straight laser.
    • Sprite name (texture) - This is the sprite used as the laser texture. The default sprite is sprites/laserbeam.spr.
    • End sprite (EndSprite) - This sprite will be placed at the end of the laser strike.
    • Texture scroll rate (TextureScroll) - This is the scroll rate of the sprite texture. Adjust as needed.
    • Starting frame (framestart) - This allows you to start the sprite on a different frame in the event that you're using a number of env_lasers in the same area. This will prevent them from looking the same.
    • Damage / second (damage) - If you want your laser to do damage (and who wouldn't?), give it a damage value here.

    Flags
    There are a number of flags you can use on the env_laser entity.

    • Start On (1) - If the env_laser has a Name, it will start off by default. Enable this flag to make the laser start on.
    • Start Sparks (16) - If this is enabled, sparks will emit from the env_laser location.
    • End Sparks (32) - If this is enabled, sparks will emit from the target location of the laser.
    • Decal End (64) - When this is enabled, there will be a burn mark at the laser target location.
