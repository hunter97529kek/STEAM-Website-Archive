Entities
All entities, all the time.

trigger_auto
Half-Life entity

description

    The trigger_auto entity lets you trigger an event as soon as you start a level.

usage

    The trigger_auto entity contains the following properties.

    • Target (target) - This should be set to the Name of the event you wish to activate with the trigger_auto.
    • Delay before trigger (delay) - The entity will pause for this length of time before activating it's targets. It's a good idea to set this to a minimum value of 0.5 to make sure that all of the entities have spawned in a map before the trigger_auto tries to activate anything.
    • Killtarget (killtarget) - This should be set to the Name of the event you wish to remove when the trigger_auto activates its events.
    • Global State to Read (globalstate) - This is the name of the global variable (set by an env_global) that can be used to control the state of the trigger_auto entity.
    • Trigger State (triggerstate) - This is the state that the trigger_auto will send to its targets. You should normally set this to On.

      Flags
    • Remove on Fire (1) - If you plan on having the player come back to the current level eventually, but you don't want the trigger_auto to activate a second time, make sure you enable this flag.
