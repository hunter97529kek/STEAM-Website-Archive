Manually Copying Existing SDK Content

For some reason, if you would rather migrate your \sourcesdk files manually instead of using the automatic batch file, this guide will show you which files should be copied to the appropriate directory in the new file structure.

This procedure uses Counter-Strike: Source as a sample configuration. For other games/MODs, substitute the appropriate names and/or directories, as described in at the end of this document.

  1. Copy content sources to the content subdirectory
  2. Copy compiled content to the game directory
  3. Copy Content files for your other games

Copy content sources to the content subdirectory

First you need to copy any Source SDK content sources files from \sourcesdk directory to the corresponding directory in \sourcesdk_content.

Note: While copying, you should not replace existing files in the \sourcsdk_content directory. These are the Source SDK samples -- replacing these files may prevent getting updated versions of the samples.

  1. Move any custom materials sources inside:

    \Steam\SteamApps\username\sourcesdk\cstrike_sample_content\materialsrc

    to

    \Steam\SteamApps\username\sourcesdk_content\cstrike\materialsrc

  2. Move any custom model sources inside:

    \Steam\SteamApps\username\sourcesdk\cstrike_sample_content\model_sources

    to

    \Steam\SteamApps\username\sourcesdk_content\cstrike\modelsrc

  3. Move any custom .VMF files inside:

    \Steam\SteamApps\username\sourcesdk\cstrike_sample_content\maps

    to

    \Steam\SteamApps\username\sourcesdk_content\cstrike\mapsrc

Copy compiled content to the game directory

Now we need to copy the compiled game content from the \sourcesdk to the corresponding directory in \counter-strike source:

  1. Move any custom material files in:

    \Steam\SteamApps\username\sourcesdk\cstrike_sample_content\materials

    to

    \Steam\SteamApps\username\counter-strike source\cstrike\materials

  2. Move any custom model files inside:

    \Steam\SteamApps\username\sourcesdk\cstrike_sample_content\models

    to

    \Steam\SteamApps\username\counter-strike source\cstrike\models

  3. Move any .BSP files inside:

    \Steam\SteamApps\username\sourcesdk\cstrike_sample_content\maps

    to

    \Steam\SteamApps\username\counter-strike source\cstrike\maps

Configure your other games

You can follow the same instructions for other games or MODs to update their game configurations.

For Half-Life 2: Deathmatch:

  • Substitute\half-life 2 deathmatch for \counter-strike source
  • Substitute \halflife 2 deathmatch\hl2mp for \counter-strike source\cstrike
  • Substitute \hl2mp_sample_content for \cstrike_sample_content

For Half-Life 2:

  • Substitute\half-life 2 for \counter-strike source
  • Substitute \halflife 2\hl2 for \counter-strike source\cstrike
  • Substitute \sdkcontent for \cstrike_sample_content

Continuing with the directory migration

When you've finished copying files for all your game configurations, continue with the instructions by resetting with Resetting your game configuration.
