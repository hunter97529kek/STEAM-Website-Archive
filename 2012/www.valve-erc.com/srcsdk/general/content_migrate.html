Migrating SDK Content to the new SDK directory structure

With the latest SDK release, the directory structure was changed to remove the restriction of having to save your game content into the \sourcesdk directory tree. This guide instructs you how to move the contents of the old \sourcesdk directory to the new structure.

For a detailed description of the new Source SDK directory layout, see The Source SDK Files and Directory Structure.

If you have not added any of your own content sources (maps, models, levels), you do not need to follow these instructions.

Warning: These assume you already had a working configuration under the \sourcesdk directory tree. If that is not the case, see the Source SDK FAQ for information on how to get a valid working configuration. Manually editing a non-working configuration using these instructions can cause even more problems and may require re-installing the SDK to repair!

  1. Use the batch script to migrate existing SDK content
  2. Resetting your game configuration
  3. Testing your new configuration

Use the batch script to migrate existing SDK content

A command-prompt batch script file has been created to aid you in moving your files in the old \sourcesdk directories to the new game and content source directories. This needs to be done so that maps and models you've created will be in the correct directory when you use the Source SDK tools.

For some reason, if you would rather copy the files yourself, see the next section, Manually copying existing SDK content.

If you wish to see the contents of this batch file before you download and run it, look here.

To use the batch script file to copy your SDK game content:

  1. Restart Steam and launch the Source SDK from the Steam launcher to make sure you have the latest updates.

  2. Close the Source SDK Launcher and exit Steam.

  3. Right-click this link and choose Save Target As... to download the batch file.

  4. Navigate to your \Steam\SteamApps\username\sourcesdk\ directory and click Save.

  5. Open an explorer window and navigate to the \Steam\SteamApps\username\sourcesdk\ directory.

  6. Double-click the file copy_sdk_content.bat.

  7. Wait for the batch file to finish copying the files. This may take several minutes.

Next, the game configuration needs to be reset to point to these new locations.

Resetting your game configuration

Now that you've copied the necessary files, you must update your game configuration to use the new directory structure. This simply consists of resetting your game configuration and having the Source SDK rebuild it automatically.

If you HAVE NOT created a MOD through the "Create A Mod" wizard:

  1. Open the SDK Launcher application from the Steam Play Games panel.

  2. Choose "Reset Game Configurations" from the SDK Launcher.

  3. Click "Reset" to reset you game configurations to the defaults.

If you HAVE created a MOD through the "Create A Mod" wizard, see the following page on Retaining a Custom MOD Game Configuration.

Testing your configuration

Now that the game configuration is reset, you can test your changes:

  1. Select a game as the Current Game in the Source SDK Launcher.

  2. Launch Hammer Level Editor to test your configuration.
