Setting Up the SDK Development Environment

Before you can begin to compile and use the Source SDK, you'll need to setup an environment in which you can do so. The SDK relies on certain environment variables to be in place so that it may infer the location of files and content during the compilation of various assets. It also requires certain updates and upgrades to software packages you may be using for it to properly compile the binary components of the Source code.


Setting the VPROJECT Environment Variable

The VPROJECT environment variable is used by many of the Source compilation tools to determine the ultimate destination or initial location of the assets it is dealing with. Failure to set this environment variable will result in failure to compile certain assets. The VPROJECT variable should point to the location of your MOD's game directory (i.e. "C:\Projects\MyMod"). This is also the directory in which the gameinfo.txt file resides for your MOD. To set the variable, follow the steps below:

WinXP

  1. Right-click on "My Computer" and choose "Properties".

  2. Select the "Advanced" tab from the dialog.

  3. Select "Environment Variables" from this tab.

  4. Under the "User Variables" section, select "New".

  5. In the dialog, under "Variable Name", add the text: VPROJECT

  6. In the "Variable Value" section, add the text: %sourcesdk%\cstrike
    Note: For this example the VPROJECT variable is pointing to the Counter-Strike game directory in Steam.
    This value should point to whatever location your MOD gameinfo.txt file is located in (i.e. "C:\Projects\MyMod", etc)

  7. Click "Ok" in all dialogs to close them.

Command Prompt

  1. Open a command line dialog.

  2. Execute the following line: set VPROJECT="C:\Projects\MyMod" (Note: The directory given should be the location of you MOD's gameinfo.txt file)
