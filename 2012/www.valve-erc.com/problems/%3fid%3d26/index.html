Problem

    When I try to compile my map, I get an error stating "missing '[ in texturedef".

Solution

    This error occurs when you try to compile a map that was worked on in Worldcraft 3.x, then taken back to Worldcraft 2.x. The compile tools think the map is of the new format, since Worldcraft 3.x inserts "mapversion" "220" in the map properties. To fix this, load the .map into any text editor and delete the line that says "mapversion" "220".

    This error can also occur if you've got a brush face that has no texture information on it. This can happen when you use prefabs from the Old Stuff prefab library (which is included with Worldcraft) without applying a texture. There may be other ways that this can occur - it is being looked into now.

      Note: Another texture name related cause is if a texture name has a space in it. This is not allowable.

    A simple fix in this case is to save your level as a .MAP file, then reload it back into Worldcraft. Worldcraft will report "There was a problem loading the MAP file. Do you want to view the error report?" If you select Yes you will likely be shown an empty error list. After this, the map will load and you will be able to compile it . You can also re-save the .MAP file as an .RMF file at this point.
